## SECTION 5: FEATURED PROJECTS

### Project 1: Advanced ML Trading System

An ensemble deep learning system for algorithmic trading combining four distinct architectures working in concert. A Temporal Fusion Transformer handles time-series forecasting across multiple timeframes. A Convolutional Neural Network performs visual chart pattern recognition, replicating how a human trader would visually identify setups. A Graph Neural Network models market structure by representing price levels, order blocks, and liquidity zones as nodes in a graph, learning which relationships matter most through attention mechanisms. Finally, a PPO reinforcement learning agent optimizes trade execution — deciding position sizing, entry timing, and exit strategy based on the outputs of the other three models.

The system was trained on over 10,000 labeled chart images and backtested using walk-forward validation across multiple market regimes from 2021 to 2024, specifically designed to avoid overfitting to any single market condition.

**Tech Stack:** PyTorch, PyTorch Forecasting, PyTorch Geometric, Stable-Baselines3, FastAPI, TimescaleDB, Redis, Docker

**Status:** Ongoing personal project, actively being refined

### Project 2: Odus — AI Linux Desktop Mentor

Odus is an AI-powered desktop assistant for Linux that fundamentally changes how troubleshooting works. Instead of searching Stack Overflow or forums, Odus captures your screen, analyzes the visible problem using Google's Gemini Vision models, and generates a sandboxed, one-click fix.

The system uses an automatic model escalation strategy — starting with Gemini 2.5 Flash for speed, and escalating to Gemini 2.5 Pro when confidence is low, balancing cost and accuracy intelligently. Safety was a primary design concern: I built a three-tier command safety engine with 18 medium-risk rules and 14 critical command blockers, ensuring the AI never executes something destructive without explicit safeguards. The entire system integrates natively with GNOME Shell, activated through a global hotkey, with compositor-level overlays for a seamless desktop experience.

**Tech Stack:** Python, PyQt6, Gemini 2.5 Flash/Pro, GNOME Shell Extension APIs, DBus, systemd, pytest

**Status:** Solo project, fully functional

### Project 3: PrismNet — Compressed RT-DETR for Edge AI

PrismNet tackles a real engineering constraint: how do you run sophisticated object detection models on edge devices with limited compute? Using RT-DETR as the base architecture, I applied L1 unstructured pruning to remove redundant weights, followed by INT8 post-training quantization, and finally compiled the optimized model using NVIDIA TensorRT for FP16 inference acceleration.

The results were significant: model size dropped by 49.7%, from 130MB to 66MB. Inference latency dropped from 133.68ms to 21.06ms — a 6.3x speedup. Frame rate improved from 7 FPS to 47.49 FPS. Remarkably, accuracy barely moved, with mAP score dropping only from 0.534 to 0.528 on the COCO benchmark. I also built a real-time GPU telemetry dashboard using Streamlit to monitor inference performance live.

**Tech Stack:** PyTorch, TensorRT, CUDA, pycocotools, Streamlit, torch.nn.utils.prune

**Status:** Solo project, complete with full benchmarking suite

### Project 4: FinChat — Financial Document RAG System

FinChat is a Retrieval-Augmented Generation system purpose-built for financial document analysis. Users can upload financial PDFs and have natural conversations with the documents, receiving answers grounded in the actual source material with citation links back to specific pages.

The retrieval architecture uses a hybrid approach — combining FAISS for dense semantic embeddings with BM25 for sparse keyword-based retrieval, then applying cross-encoder reranking to surface the most contextually relevant chunks before generation. Inference runs locally through llama.cpp, keeping the system fast and cost-effective. I worked alongside my teammate Armaan Choudhary on this project, primarily contributing the RAG pipeline implementation, retrieval and indexing logic, and backend integration work.

**Tech Stack:** FastAPI, sentence-transformers, FAISS, BM25, llama.cpp, Tailwind CSS

**Status:** Team project with Armaan Choudhary, functional MVP
